# Vitae — Agent Guidelines

Vitae is a minimal Markdown-based resume generator. It uses **Vite + TailwindCSS v4** to render a resume from `src/resume.md` into a styled HTML page and auto-generates a PDF via Puppeteer on build.

## Project Structure

```
src/resume.md          # Resume content (edit this to update the resume)
src/styles/index.css   # TailwindCSS v4 styles
core/plugin.ts         # Vite plugin — transforms Markdown → HTML, generates PDF
core/index.ts          # PDF builder & markdown-it config
core/themes.ts         # 8 built-in color themes with light/dark CSS variables
core/main.ts           # Entry point
index.html             # HTML shell (uses #[title] and #[content] placeholders)
vite.config.ts         # Configure pdfName, webTitle, theme, pdfMargin
```

## Key Conventions

### Resume Content
- All resume copy lives in `src/resume.md`. This is the only file users need to edit for content changes.
- The Vite plugin reads this file at build/dev time and injects rendered HTML into `index.html` via the `#[content]` placeholder.

### Themes
- Themes are defined in `core/themes.ts` as `ThemeConfig` objects with `colors` (light) and optional `darkColors` (dark mode).
- Each theme exposes CSS custom properties under `--theme-*` namespace (e.g. `--theme-primary`, `--theme-accent`, `--theme-bg-page`).
- Always use `--theme-*` variables in `src/styles/index.css` — never hardcode color values.
- To add a new theme: add it to the `Theme` union type, implement a `ThemeConfig` entry in the `themes` record, and export from `core/themes.ts`.

### PDF Generation
- PDF is generated by Puppeteer in `core/index.ts` via `pdfBuilder()`.
- PDF generation is **skipped** when `CI=true` or `VERCEL=true` (set automatically on Vercel/GitHub Actions).
- Chrome/Chromium path is resolved from `PUPPETEER_EXECUTABLE_PATH` env var or `puppeteerExecutablePath` plugin option; falls back to common OS paths.

### Plugin Options (`vite.config.ts`)
| Option | Type | Description |
|---|---|---|
| `pdfName` | `string` | Output PDF filename (no extension) |
| `webTitle` | `string` | HTML `<title>` tag |
| `theme` | `Theme` | One of the 8 built-in theme names |
| `pdfMargin` | `number \| object` | Puppeteer page margin |
| `markdown` | `(md) => void` | Extend markdown-it with plugins |

## Dev Workflow

```bash
pnpm dev    # Start dev server with hot reload
pnpm build  # Build HTML + PDF to dist/
```

## Constraints

- Do **not** modify `dist/` — it is build output.
- Do **not** hardcode colors in CSS; always use `--theme-*` variables so themes work correctly.
- Keep `core/` logic framework-agnostic — no UI framework imports.
- TypeScript strict mode is on; avoid `any` unless necessary.
